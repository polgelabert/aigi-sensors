PPDU: Physical Protocol Data Unit -> Trama = entre 0 i 127 bytes

Trama + petita que Eth (1500 bytes) -> SW, HW millor processat, + simple i EFICIENT


Capa MAC:

  • Gestiona l'accés al medi:
    • Beacon: Sincronització, no hi ha retard, estructura Supertrama
    • Sense: Retard, CSMA/CA
  • Fiabilitat: Opcional (AKN)
  • Seguretat: Sí
  • Supertrama: 15-245s, B | CAP | CFP | Idle Period
  • Connexió a Internet: Sí

6LoWPAN


IPv6 Low Power WPAN

Mecanismes:

  • Autoconfiguració d'adreces:
    • Prefix xarxa: Global/Local
    • Interface ID: MAC 802.15.4 (Llarga/curta)
  • Compressió capçaleres i fragmentació
    • IpV6: 1280 bytes
    • 802.15.4: 127 bytes
  • Format trama
  • Optimització Neigbor Discovery

RPL

Neix de la necessitat d'utilitzar un protocol d'encaminament route-over basat en IP

IPv6 Routing Protocol for Low Power and Lossy Networks

Tipologia DAG (graf acíclic) optimitzat per a:

  • MP2P - Multipoint to Point
  • P2MP - Point to Multipoint
  • P2P - Point to Point

Els nodes envien periòdicament missatges DIO (Dodag Info Object = "Hello")

Incorporació d'un node: quan coneix la info (rep DIO/envia DIS), determina els seus veins i designa Default Parent

Objective Function (OF) -> Mitjantçan mètrica i restriccions, permet triar a un node el seu Default Parent.


Rutes:

  • Upward: Node envia info al Def. Parent
  • Downward: Nodes envien DAO (Dodag Adv. Object) al Def. Parent (Akn opt) fins l'arrel, que:
    • Storing Mode: Els nodes intermitjos guarden taula d'encaminament (Destí, Next Hop)
    • Non Storing Mode: Només l'arrel té guarda la taula. En la capçelera hi han els nodes intermitjos.

CoAP


HTTP -> Inadecuat per restriccions de xarxa:

  • Missatges massa llargs
  • Massa BW, memòria, energia sensors i no suporta sleep nodes

CoAP (Constrained Application Protocol) es basa en UDP:

  • Model REST (Servidor - Client)
    • Petició - Resposta: GET, POST, PUT, DELETE
    • Missatges: CON, NON, AKN, RST

Format missatges:

  • Capçalera entre 4bytes i 32 bytes (max)
  • Missatge de 16bytes


Tipus de resposta

  • Piggybacked: Akn + Data -> = missatge
  • eparada: Akn i Data -> different missatge


Caché que guarda peticions (no ha de demanar-la més)

Proxy: Agent substitutiu que actua en nom d'una altra persona/entitat. Pot fer de Servidor o Client

Oberve: Si un node es vol subscriure a un servidor que notifiqui canvis del seu recurs -> GET (Observe)
